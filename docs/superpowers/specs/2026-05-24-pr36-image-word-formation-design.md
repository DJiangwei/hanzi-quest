# PR #36 — Image-prompted word formation (text-stimulus, image-ready)

**Date:** 2026-05-24
**Author:** Claude + David (brainstorm + re-scope)
**Status:** Spec — pending implementation plan

---

## 1. Goal

Add a new scene type `image_word` where Yinuo sees a kid-friendly description of a 词's meaning + the base 字 she's learning, then picks the correct 词 (multi-char word containing that 字) from 4 candidates. The description is generated by DeepSeek as part of the existing AI scene-gen pipeline. **No real image generation in this PR** — the scene renders a stylized "description card" that **will be swapped for a Flux-generated image in PR #37** when Vercel Blob + AI Gateway are provisioned.

This is a deliberate two-step ship:
- **PR #36** (this one): scene mechanic + AI pipeline + DB column + compile slot + backfill. Pure DeepSeek; zero new infra.
- **PR #37** (later): swap the description card for a real image when Vercel Blob + AI Gateway are wired. **1-line scene change + 1-column schema add.** The compile-week logic, scene config schema, and route plumbing stay identical.

After PR #36 ships:
- Each ≥10-char week's practice grows from 12 → 14 scenes (added: 2 image_word in sight).
- New weeks auto-generate per-word imageHooks during publish (no new infra; same DeepSeek call returns extra structured field).
- The 10 already-published weeks get imageHooks backfilled by a one-off script (~$0.10 total DeepSeek cost).
- Boss-unlock threshold bumps 6 → 7 to keep the proportion ≈ 50% of practice cleared.
- Words without an imageHook (e.g. DeepSeek timeout, schema rejection) gracefully fall back to extra `visual_pick`.

## 2. Locked decisions (from brainstorm + re-scope)

| Decision | Choice |
|---|---|
| Stimulus type | **Text "description card"** for PR #36. AI-generated image swap deferred to PR #37 once Vercel Blob + AI Gateway are provisioned. |
| Description source | **DeepSeek extends WeekContent V2** — each word gets an `imageHook` string field (max 120 chars, kid-friendly, single-subject). Same model + API call shape as today; no new provider. |
| Scene mechanic | **DescriptionCard + base 字 → pick 词** from 4 options, preferring distractors that contain the same base 字. |
| Description persistence | **New `words.image_hook` text column** (nullable) (migration 0012). Compile reads this; renders the card when present, falls back when null. |
| Compile placement | **+2 `image_word` slots in sight segment.** Practice grows 12 → 14 per full week. |
| Boss inclusion | **No.** Boss rotation stays at 5 types — image_word stays exclusive to practice. |
| Boss unlock threshold | **Bump 6 → 7** (~50% of new 14-scene practice). |
| Backfill | **One-off `scripts/backfill-word-image-hooks.ts`** — fills NULL `image_hook` for the ~300 existing prod words via a single DeepSeek call per word. Idempotent. |
| Future image swap (PR #37) | Add `words.image_url` column; scene renders `<img>` when set, falls back to DescriptionCard otherwise. No schema-redesign required; both columns coexist. |

## 3. Architecture

**Scene component** `src/components/scenes/ImageWordScene.tsx`:

```tsx
'use client';

import { useMemo } from 'react';
import { MultipleChoiceQuiz } from './MultipleChoiceQuiz';
import { shuffle } from '@/lib/scenes/sample';

interface WordOption {
  wordId: string;
  text: string;
  imageHook: string | null;
  meaningEn: string | null;
}
interface BaseChar { characterId: string; hanzi: string }

interface Props {
  baseChar: BaseChar;
  correctWord: WordOption;          // imageHook must be set
  distractors: WordOption[];        // 3 wrong words
  onComplete: (correct: boolean) => void;
}

export function ImageWordScene({ baseChar, correctWord, distractors, onComplete }: Props) {
  const choices = useMemo(
    () =>
      shuffle([correctWord, ...distractors]).map((w) => ({
        key: w.wordId,
        label: <span className="text-3xl font-extrabold">{w.text}</span>,
        isCorrect: w.wordId === correctWord.wordId,
      })),
    [correctWord, distractors],
  );

  return (
    <MultipleChoiceQuiz
      prompt={
        <span>
          看图选词 / Pick the word with{' '}
          <span className="ml-1 rounded-md bg-amber-200 px-2 py-0.5 text-2xl font-extrabold text-amber-900">
            {baseChar.hanzi}
          </span>
        </span>
      }
      stimulus={
        <div className="flex h-48 w-72 items-center justify-center rounded-2xl border-4 border-amber-800/30 bg-gradient-to-br from-amber-50 via-sky-50 to-amber-50 p-5 text-center shadow-lg">
          <span className="text-3xl" aria-hidden>
            ✨
          </span>
          <p className="ml-3 text-base font-semibold leading-snug text-amber-950">
            {correctWord.imageHook ?? correctWord.meaningEn ?? correctWord.text}
          </p>
        </div>
      }
      choices={choices}
      onComplete={onComplete}
    />
  );
}
```

### 3.5 Compile changes

`src/lib/scenes/compile-week.ts` — extend `PracticeSizing` and the sight emission:

```ts
interface PracticeSizing {
  audio: number;
  sight: number;       // image_pick + visual_pick + word_match slots
  imageWord: number;   // NEW
  meaning: number;
}

function computePracticeSizing(n: number): PracticeSizing {
  if (n < 2)  return { audio: 0, sight: 0, imageWord: 0, meaning: 0 };
  if (n < 4)  return { audio: 1, sight: 1, imageWord: 1, meaning: 4 };  // 7 practice
  if (n < 10) return { audio: 2, sight: 2, imageWord: 1, meaning: 6 };  // 11 practice
  return { audio: 3, sight: 3, imageWord: 2, meaning: 6 };              // 14 practice
}
```

In the sight segment emission, after the existing image_pick / visual_pick / word_match block, add:

```ts
const imageWordId = tmplByType.get('image_word');
if (imageWordId && sizing.imageWord > 0) {
  const eligibleChars = chars.filter((c) =>
    c.words.some((w) => w.imageHook !== null),
  );
  const visualId = tmplByType.get('visual_pick');
  let emitted = 0;
  for (let slot = 0; slot < sizing.imageWord; slot++) {
    if (eligibleChars.length === 0) break;
    const target = pickRandom(eligibleChars);
    const eligibleWords = target.words.filter((w) => w.imageHook !== null);
    const correctWord = pickRandom(eligibleWords);
    const allOtherWords = chars
      .flatMap((c) => c.words.filter((w) => w.id !== correctWord.id))
      .map((w) => w.id);
    const distractors = shuffle(allOtherWords).slice(0, 3);
    if (distractors.length < 3) break;
    push(
      imageWordId,
      { characterId: target.id, wordId: correctWord.id, distractorWordIds: distractors },
      'sight',
      `practice:image_word:${slot}`,
    );
    emitted++;
  }
  // Fallback: unfilled slots become extra visual_pick to keep practice count steady
  if (emitted < sizing.imageWord && visualId) {
    for (let i = emitted; i < sizing.imageWord; i++) {
      const target = pickRandom(chars);
      push(
        visualId,
        { characterId: target.id },
        'sight',
        `practice:visual_pick:fallback-${i}`,
      );
    }
  }
}
```

**Updated practice totals:**

| chars in week | review | practice (audio+sight+imageWord+meaning) | boss | total levels |
|---|---|---|---|---|
| ≥ 10 | N | **14** (3+3+2+6) | 1 | N+15 |
| 4–9  | N | 11 (2+2+1+6) | 0 | N+11 |
| 2–3  | N | 7 (1+1+1+4) | 0 | N+7 |
| < 2  | N | 0 | 0 | N |

(Yinuo's 8-char weeks #9–#10 grow from N+10 → N+11 levels; her 10-char weeks #1–#8 grow from N+13 → N+15.)

**`BOSS_UNLOCK_PRACTICE_THRESHOLD = 7`** (was 6). One-line update to `src/lib/scenes/configs.ts`.

### 3.6 Pool + SceneRunner data wiring

`SceneRunner`'s `pool` items today carry `firstWord: string | null`. We need each character's full word list (with `imageHook`) for image_word distractor lookup at runtime.

Extend the `CharacterDetail` type in `SceneRunner.tsx`:

```ts
interface CharacterWord {
  id: string;
  text: string;
  imageHook: string | null;
  meaningEn: string | null;
}
interface CharacterDetail {
  characterId: string;
  hanzi: string;
  pinyinArray: string[];
  meaningEn: string | null;
  meaningZh: string | null;
  imageHook: string | null;       // existing — char-level hook
  words: CharacterWord[];         // NEW — full per-char word list
  sentence: { id: string; text: string; translationEn: string | null } | null;
}
```

The two section pages (`level/[weekId]/page.tsx` legacy + `level/[weekId]/[section]/page.tsx`) build the pool from `getCharactersWithDetailsForWeek(weekId)` (which already returns `words: WordRow[]` per character). Just map them into the new shape:

```ts
const pool = characters.map((c) => ({
  characterId: c.id,
  hanzi: c.hanzi,
  pinyinArray: c.pinyinArray ?? [],
  meaningEn: c.meaningEn ?? null,
  meaningZh: c.meaningZh ?? null,
  imageHook: c.imageHook ?? null,
  words: c.words.map((w) => ({
    id: w.id,
    text: w.text,
    imageHook: w.imageHook ?? null,
    meaningEn: w.meaningEn ?? null,
  })),
  sentence: c.sentence ? { ... } : null,
}));
```

`firstWord` is removed; nothing should reference it after this change (verify via grep during plan stage). If it's used elsewhere, derive on the fly: `c.words[0]?.text`.

**RSC boundary safety**: `pool` items are pure JSON — no functions, no React components — so they cross the server → client boundary fine. Per the CLAUDE.md landmine: not applicable.

### 3.7 Backfill script

**`scripts/backfill-word-image-hooks.ts`** (one-off, idempotent):

```ts
/**
 * Backfill words.image_hook for words where it's NULL.
 *
 * For each word in batched concurrency=10:
 *   1. Call DeepSeek with a one-shot prompt: "Write a kid-friendly visual
 *      description of the Chinese word X (meaning Y). Single subject, vivid,
 *      no proper nouns, no text in scene. Max 120 chars."
 *   2. UPDATE words SET image_hook = ... WHERE id = ... AND image_hook IS NULL.
 *
 * Idempotent — skips words that already have image_hook set.
 * Re-runnable after partial failure (retries NULLs only).
 *
 * Usage: pnpm tsx scripts/backfill-word-image-hooks.ts
 * Cost: ~$0.0003 per word via DeepSeek V4 Pro = ~$0.10 total for ~300 words.
 *
 * CAUTION: writes to prod via shared DATABASE_URL on Neon free tier.
 */
```

Wall time: ~30s at concurrency=10 (~1s per DeepSeek call). Failure mode: re-run only retries NULLs.
